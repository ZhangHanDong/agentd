# P0.9 Deployment Checklist — the real-environment capstone

P0.9 was completed **code-complete + contract-tested in-process** (see
[`2026-05-29-agentd-p0.9-capstone.md`](./plans/2026-05-29-agentd-p0.9-capstone.md)).
The offline build box has **no real AI agents, no mempal server, no tmux session,
no docker/synapse, and cannot SIGKILL itself mid-assertion**, so the items below —
everything that needs a real process boundary or external service — are delivered
as a runnable checklist to execute on a real machine. None of them is new logic;
each exercises an already-tested seam against a real counterpart.

Legend: **[proven-by-test]** = the contract is verified in-process; this step adds
only environment coverage.

## 0. Prerequisites (real box)

- [ ] `tmux` on `PATH`; a terminal multiplexer session available.
- [ ] At least one real coding-agent CLI (Claude Code / codex / gemini) on `PATH`.
- [ ] `gh` authenticated (`gh auth status`) or `GH_TOKEN` exported — for `execute.dot`'s `open_pr`.
- [ ] `agent-spec` on `PATH` — for `draft.dot`'s `lint_spec` and `execute.dot`'s `verify_lifecycle`/`draft_plan`.
- [ ] (Optional) a running mempal MCP server if you want non-empty memory; otherwise the daemon's `OfflineMempal` returns empty (workflows don't require it).

## 1. Boot the daemon

- [ ] `cargo run -p agentd-bin --bin agentd -- --db-path ./agentd.db --port 8787 --workflows-dir ./workflows`
- [ ] Confirm the log line `agentd daemon listening` on `127.0.0.1:8787` and the in-flight-runs count.
- [ ] `curl -s localhost:8787/healthz` → `ok`. **[proven-by-test]** (p96 via oneshot)

## 2. Live run start (real agents) — the rmcp wire

- [ ] `cargo run -p agentctl -- run start --flow draft --daemon-url http://127.0.0.1:8787 <issue-id>`
      → daemon creates the run, parks at `propose_spec`, returns `{run_id, status:"parked"}`. **[proven-by-test]** (p95 POST /runs)
- [ ] **Real-agent stdio smoke:** run `bash scripts/agentd_real_claude_smoke.sh --dry-run`, then `bash scripts/agentd_real_claude_smoke.sh --preflight-only`, then opt in with `AGENTD_REAL_CLAUDE_SMOKE=1 bash scripts/agentd_real_claude_smoke.sh --execute`. The harness starts the daemon, triggers `draft.dot`, waits for the run to leave `propose_spec`, and writes `issue.md`, `preflight.log`, `daemon.log`, `agentctl.out`, `run_snapshot.json`, `events.snapshot`, and `summary.txt` under its state dir. P122 version-aligns `rmcp` and makes the stdio boundary speak the MCP lifecycle/tool result shapes; P123 makes the Claude Code launcher generate a per-agent MCP config for `agentd mcp-stdio` and start Claude with `--mcp-config ... --strict-mcp-config`; P124 makes the real authenticated Claude smoke repeatable and auditable. The tool layer + `open_task` + emit are **[proven-by-test]** (p90) — only the real agent process and paid/authenticated CLI call are new.
- [ ] Watch progress: `curl -s localhost:8787/runs/<run_id>` and the SSE stream `curl -s localhost:8787/runs/<run_id>/events`. **[proven-by-test]** (p73/p96)

## 3. execute.dot end-to-end (3 reviewers + open_pr)

- [ ] Freeze the draft (local file convention `.agentd/run/frozen.spec.md`, Path-B Δ1), then `run start --flow execute <spec-id>`.
- [ ] Confirm the run parks at `implement` (one agent), then `review` (three reviewer agents: claude-sec / codex-perf / gemini-readability spawned blind over the frozen bundle). Each reviewer learns its `review_run_id` from its **spawn context** (the piece deferred from the offline harness).
- [ ] `aggregate` (majority_pass) → `open_pr` shells `gh pr create --fill` → a real PR opens. **[proven-by-test]** the walk + the `gh` argv (P0.8 + p90); only the real `gh`/token is new.
- [ ] The goal_gate recovery edge (P0.8 D5) is exercised if a gate is unmet.

## 4. kill -9 disaster-recovery drill (real SIGKILL)

- [ ] With a run parked, `kill -9 <agentd-pid>`.
- [ ] Re-launch the daemon with the **same** `--db-path`. Confirm the in-flight-runs log and that delivering the next agent event resumes the run from its checkpoint — no completed node re-runs, the event `seq` continues. **[proven-by-test]** the resume + idempotent-replay contract (p92, simulated restart); only the real SIGKILL is new.
- [ ] Modify a workflow `.dot`, then attempt resume → `WorkflowShaChanged`; re-run with `--accept-workflow-change` (operator-gated). **[proven-by-test]** (p92 `resume_guard` both branches). *(Wire `--accept-workflow-change` into the resume path when the rmcp/boot-resume lands.)*

## Known gaps to close in the real-env wiring (not P0 blockers)

- **TaskAssignment** (P0.9 D5): P121 resolves the old `agent_id` gap by persisting task-run ownership in `task_runs.agent_id` and returning it from production `open_task`; the remaining gaps are `spec_path`/`plan_path`/`context_pack`, which still come from spawn context or a per-run file convention rather than the store.
- **Initial run context** (P0.9 9c): `start_workflow` accepts but does not seed `context`; real runs inject `issue_id`/`spec_path` via the spawn context or a per-run file convention.
- **Checkpoint/outcome non-atomicity** (advisor): a crash between the outcome insert and the checkpoint write can leave a duplicate-able node; replay-safety holds (closed park → Ignored), but same-transaction emit is a P1 hardening.
- **SSE field sanitization** (P0.7 D9): when emit payloads can contain CR/LF, sanitize at the SSE boundary (the emit point uses compact JSON, which avoids it for the current event kinds).
- **Re-park emit noise** (P1) — ✅ RESOLVED (`6f41188`): the emit fired on every state-changing `RunProgress`, so a re-park at the SAME node (e.g. a fan-out review collecting verdicts) emitted a fresh `run_parked` per non-final verdict — truthful but noisy for an SSE dashboard. Fixed by deduping same-node re-parks at the emit point (suppress when the run's most recent event is already a `run_parked` for that node — `event_repo::last`); the `goal_gate_unmet → implement` re-loop still emits because a `review` park intervenes. See `specs/e2e/p90-runhost-contract.spec.md`.
