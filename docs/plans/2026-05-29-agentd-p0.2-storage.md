# P0.2 — Storage Layer Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans. Follows P0.0 conventions and the P0.1 TDD pattern.

**Goal:** Implement `agentd-store` — the sqlx-backed SQLite layer that materializes the 14 tables from design §3.3 and exposes five repositories used by the engine (`projects`, `runs`, `node_outcomes`, `artifacts`, `human_waits`, with secondary roles for `issues`, `task_runs`, `review_runs`, `review_verdicts`, `mempal_outbox`, `matrix_events`, `events`, `agents`, `project_agents`, `schema_meta`). After this phase the engine's port traits (defined in P0.1) gain real implementations and a real `runs/` directory layout under `~/.agentd/`.

**Architecture:** `agentd-store` provides a `SqliteStore` that implements the single `agentd_core::ports::Store` trait defined in P0.1 Task 6.5 (one `#[async_trait]` trait carrying `insert_run` / `update_run_status` / `insert_node_outcome` / `latest_outcome` / `count_attempts` / `open_human_wait` / `answer_human_wait` / `insert_review_run` / `insert_review_verdict` / `count_verdicts` / `write_checkpoint` / `load_checkpoint`). P0.1 deliberately did NOT split this into per-domain sub-traits — there is one trait. P0.2 may ADD methods this phase needs (issues, artifacts, task_runs, mempal_outbox, matrix_events, events) either by widening `ports::Store` (preferred — keep one trait) or by exposing inherent `SqliteStore` methods for surfaces the engine doesn't call. Migrations live under `crates/agentd-store/migrations/` as numbered SQL files; sqlx-cli is the dev-time tool but at runtime the daemon embeds migrations via `sqlx::migrate!()`. Connection pool is `SqlitePool` (`tokio` runtime). All writes use prepared statements; serde JSON columns are typed via `serde_json::Value`.

**Tech Stack:** `sqlx = { workspace = true }` (sqlite + macros + migrate + runtime-tokio), `directories = "5"` for `~/.agentd/` resolution, `tempfile` (dev).

**Spec:** Five contracts under `specs/store/`:
- `p20-migrations-and-schema.spec.md`
- `p21-run-and-outcome-repository.spec.md`
- `p22-artifact-content-addressed.spec.md`
- `p23-human-wait-repository.spec.md`
- `p24-mempal-outbox-persistence.spec.md`

**Estimate:** 140 agent-rounds.

**Deps:** P0.0, P0.1 (port traits to implement).

**Exit criteria:**

- `cargo test -p agentd-store` exits 0 (21 scenarios across 5 specs: migrations 4 + runs/outcomes 4 + artifacts 4 + human_waits 4 + outbox 5).
- `sqlx migrate run` against an empty file applies all 14 tables exactly once; rerun is a no-op; `schema_meta.version = '1'`.
- A binary smoke test (`cargo run --example wire_real_store -p agentd-bin`) starts the daemon stub, opens `~/.agentd/agentd.db`, applies migrations, writes a Project row, exits cleanly.
- `agent-spec lifecycle 'specs/store/*.spec*' --code . --min-score 0.7` passes for all five specs.

```rust
//! Open + migrate the SQLite DB. Enables WAL + foreign_keys.

use crate::error::StoreError;
use sqlx::sqlite::{SqliteConnectOptions, SqlitePoolOptions};
use sqlx::{ConnectOptions, SqlitePool};
use std::path::Path;
use std::str::FromStr;

pub static MIGRATIONS: sqlx::migrate::Migrator = sqlx::migrate!("./migrations");

pub async fn open(db_path: &Path) -> Result<SqlitePool, StoreError> {
    let opts = SqliteConnectOptions::from_str(&format!("sqlite://{}?mode=rwc", db_path.display()))?
        .create_if_missing(true)
        .journal_mode(sqlx::sqlite::SqliteJournalMode::Wal)
        .foreign_keys(true)
        .busy_timeout(std::time::Duration::from_secs(5))
        .log_statements(tracing::log::LevelFilter::Trace);

    let pool = SqlitePoolOptions::new().max_connections(8).connect_with(opts).await?;
    MIGRATIONS.run(&pool).await?;
    Ok(pool)
}
```

- [ ] **Step 6: `error.rs`**

```rust
use agentd_core::CoreError;
use thiserror::Error;

#[derive(Debug, Error)]
pub enum StoreError {
    #[error("sqlx: {0}")]
    Sqlx(#[from] sqlx::Error),

    #[error("migrate: {0}")]
    Migrate(#[from] sqlx::migrate::MigrateError),

    #[error("serde: {0}")]
    Serde(#[from] serde_json::Error),

    #[error("not found")]
    NotFound,

    #[error("conflict: {0}")]
    Conflict(String),

    #[error("invariant violated: {0}")]
    Invariant(String),
}

// REQUIRED so `impl agentd_core::ports::Store for SqliteStore` can return the
// trait's error type (CoreError) while repos produce StoreError internally.
// Legal under the orphan rule: `From<StoreError>` has the local `StoreError`
// as its type parameter, so we may impl it for the foreign `CoreError`.
// (Fixes bug-hunt M2.)
impl From<StoreError> for CoreError {
    fn from(e: StoreError) -> Self {
        match e {
            StoreError::NotFound => CoreError::Store("not found".into()),
            StoreError::Conflict(m) => CoreError::Store(format!("conflict: {m}")),
            other => CoreError::Store(other.to_string()),
        }
    }
}
```

**Trait vs. inherent return types:** the inherent repository methods (`RunRepo`,
`OutcomeRepo`, etc.) return `Result<_, StoreError>` — that is their natural error.
The `impl agentd_core::ports::Store for SqliteStore` trait methods return
`Result<_, CoreError>` (the trait's declared error type); they call the repos and
apply `?`, which auto-converts via the `From` impl above. Do NOT change the trait's
error type to `StoreError` — agentd-core must not know about `StoreError`.

//! Spec: p20-migrations-and-schema
//! Scenarios:

//!   migrations_are_idempotent_on_rerun
//!   migrations_create_all_fourteen_tables

use agentd_store::pool;
use sqlx::Row;

async fn fresh_db() -> (tempfile::TempDir, sqlx::SqlitePool) {
    let dir = tempfile::tempdir().expect("tempdir");
    let path = dir.path().join("agentd.db");
    let pool = pool::open(&path).await.expect("open + migrate");
    (dir, pool)
}

#[tokio::test]
async fn migrations_apply_to_fresh_db() {
    let (_dir, pool) = fresh_db().await;
    let row = sqlx::query("SELECT value FROM schema_meta WHERE key = 'version'")
        .fetch_one(&pool).await.expect("query schema_meta");
    let v: String = row.get(0);
    assert_eq!(v, "1");
}

#[tokio::test]
async fn migrations_are_idempotent_on_rerun() {
    let (dir, pool1) = fresh_db().await;
    drop(pool1);
    // Re-open the same path — migrations should not re-apply or error.
    let path = dir.path().join("agentd.db");
    let pool2 = pool::open(&path).await.expect("reopen + re-migrate");
    let row = sqlx::query("SELECT COUNT(*) AS c FROM schema_meta")
        .fetch_one(&pool2).await.expect("query");
    let c: i64 = row.get(0);
    assert_eq!(c, 1, "schema_meta should still have exactly one row");
}

#[tokio::test]
async fn migrations_create_all_fourteen_tables() {
    let (_dir, pool) = fresh_db().await;
    let names: Vec<String> = sqlx::query_scalar(
        "SELECT name FROM sqlite_master WHERE type='table' AND name NOT LIKE 'sqlite_%' AND name NOT LIKE '_sqlx%'"
    ).fetch_all(&pool).await.expect("list tables");
    let expected: &[&str] = &[
        "projects", "agents", "project_agents", "issues", "runs",
        "node_outcomes", "artifacts", "task_runs", "review_runs",
        "review_verdicts", "human_waits", "mempal_outbox",
        "matrix_events", "events", "schema_meta",
    ];
    for t in expected {
        assert!(names.iter().any(|n| n == t), "missing table: {t}");
    }
    // 15 names because schema_meta + 14 design tables; design §3.3 says "14 tables"
    // but schema_meta is counted as housekeeping. We assert >= 15 here.
    assert!(names.len() >= 15, "expected ≥ 15 tables (14 design + schema_meta), got {}", names.len());
}

#[tokio::test]
async fn migrations_set_schema_meta_version_to_1() {
    let (_dir, pool) = fresh_db().await;
    let v: String = sqlx::query_scalar("SELECT value FROM schema_meta WHERE key='version'")
        .fetch_one(&pool).await.expect("query");
    assert_eq!(v, "1");
}

use agentd_store::repos::projects::ProjectRow;
use agentd_store::{paths, pool, SqliteStore};

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    tracing_subscriber::fmt::init();
    paths::ensure_layout()?;
    let p = pool::open(&paths::db_path()).await?;
    let store = SqliteStore::new(p);
    // Insert and read back one project to prove writes work end-to-end.
    let row = ProjectRow {
        id: "p_smoke".into(), name: "smoke".into(), repo_path: "/tmp/smoke".into(),
        github_repo: None, matrix_room_id: None, mempal_wing: "smoke".into(),
        agentflow_dir: ".agentflow".into(), created_at: 0, updated_at: 0,
    };
    store.projects().insert(&row).await?;
    let got = store.projects().get("p_smoke").await?;
    assert!(got.is_some(), "project did not round-trip");
    tracing::info!("smoke test ok, agentd.db at {}", paths::db_path().display());
    Ok(())
}

## Cross-cutting: `.sqlx/` and compile-time query check

We deliberately do **not** use `sqlx::query!` macros in v0 — those require `DATABASE_URL` at compile time and complicate CI. Instead, use `sqlx::query_as` with `FromRow` derives where the schema is stable. This decision can be revisited in P2; revisit when the schema stabilizes.

Add to CONTRIBUTING.md (small append):

```markdown
### Storage

agentd uses sqlx in "runtime" mode — no compile-time query checking (no `.sqlx/`
metadata committed). Use `query_as::<_, RowStruct>(...)` for typed results.
We may switch to `query!` macros in a later phase once the schema is stable.
```
