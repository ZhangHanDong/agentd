# P0.3 — TmuxBackend v0 (FakeRunner-tested)

> Generated at phase start (roadmap policy) from design §4. Implements the real
> agent-spawning backend in `agentd-tmux`, driven entirely through the P0.1
> `CommandRunner` port so every test runs against a `FakeRunner` — no real tmux
> server, no spawned processes.

## Architecture decisions (locked before coding)

These resolve the design-vs-P0.1 tensions up front (advisor-reviewed):

- **D1 — Keep the agentd-core `AgentBackend` trait spawn-only; the other six
  capabilities are INHERENT methods on `TmuxBackend`.** Design §4.1 lists seven
  trait methods, but in P0.3 nothing calls `send_prompt`/`capture`/`status`/
  `interrupt`/`shutdown`/`rebind` *through the trait* — `spawn` calls
  `wait_for_ready`/`send_prompt`/`capture` internally, and `status`/`shutdown`/
  `rebind` get their first trait-level callers in P0.6/P0.7 (which will shape
  the `CaptureOpts`/`ShutdownOpts`/`ShutdownReport` structs). Widening now would
  force `FakeBackend` to grow six unexercised stubs and freeze option structs
  before a consumer exists (the inverse of the P0.2 lesson, where the engine was
  a real consumer). The P0.1 note said P0.3 "*may* widen" — permission, not
  obligation. **Net: zero edits to the tagged `agentd-core`; all P0.3 code lives
  in `agentd-tmux`.** Widen the port trait in the phase that has a trait-caller.

- **D2 — Trait/inherent error type is `CoreError`, NOT `BackendError`.** Same
  move as P0.2's `StoreError→CoreError`: `TmuxBackend` keeps a private
  `BackendError` (the rich Recoverable/PartialDelivery/Fatal/Invariant variants
  of §4.2) and maps to `CoreError::Backend(String)` at the public boundary
  (`AgentBackend::spawn` returns `CoreError`; inherent methods return
  `BackendError` internally, mapping when a public surface needs `CoreError`).

- **D3 — Inject the runner as `Arc<dyn CommandRunner>`, not a generic
  `TmuxBackend<R>`.** Matches the object-safe seam the daemon will hold; avoids
  fighting `Ports`. Tests inject `agentd_core::test_support::RecordingCommandRunner`
  (via the `test-support` dev-dependency) as the FakeRunner.

- **D4 — Assert on argv structure, not captured stdin.** `RecordingCommandRunner`
  records `{program, args}` only. That is sufficient for all seven §4.11 cases:
  the buffer path is proven by `set-buffer`/`paste-buffer` argv and the *absence*
  of the prompt payload as an arg; the large-prompt case by `set-buffer -` (the
  `-` = read-from-stdin marker) in argv. We do NOT extend the fake (keeps the
  blast radius in `agentd-tmux`).

- **D5 — Forbidden-token discipline.** The hard rule is the literal `send-keys -l`
  (payload-as-keys). The buffer path legitimately calls `send-keys -t <addr>
  Enter` (a bare key, no `-l`), so the CI/guard pattern `send-keys -l` does not
  match it. Spec `Forbidden` clauses backtick ONLY `send-keys -l`, never bare
  `send-keys`, so the guard's extracted token can't flag the legitimate Enter
  call. Code must never contain the literal `send-keys -l`.

## Task breakdown (each: spec → failing test → impl → clippy/fmt → `./scripts/check.sh` → commit)

### Task 1 — Skeleton: error, tmux discovery, config, TmuxBackend struct, real runner
- `Cargo.toml`: `agentd-core` (path), `tokio` (full), `async-trait`, `thiserror`, `tracing`; dev: `agentd-core` with `features=["test-support"]`, `tempfile`, `tokio-test`. `publish = false` (path dep / cargo-deny).
- `error.rs`: `BackendError` (§4.2 variants) + `From<BackendError> for CoreError` (orphan-rule-legal; maps to `CoreError::Backend`).
- `discovery.rs`: `tmux_bin()` per §4.4 order (`AGENTD_TMUX_BIN` > homebrew > /usr/local > /usr/bin > `which`); missing → `Fatal` with install hint.
- `config.rs`: `ready_patterns` (per-CLI, from a `[ready_patterns.*]` table; defaulted, overridable) + `inject_delay` (60ms) + timeouts.
- `runner.rs`: `TokioCommandRunner` implementing `CommandRunner` via `tokio::process::Command` (cwd/env/stdin/timeout from `RunOpts`; maps exit code to `CommandOutput`, launch/timeout to `CommandError`). This is the production runner the daemon wires (design D6).
- `backend.rs`: `TmuxBackend { runner: Arc<dyn CommandRunner>, tmux_bin: PathBuf, cfg: Config }` + `new`; private `tmux(args, opts)` helper that prefixes `tmux_bin`.
- Tests: `tmux_bin` honors `AGENTD_TMUX_BIN`; missing-tmux → `Fatal` with the install hint string. **(§4.11 case 6.)**

### Task 2 — `spawn` (the only trait method) + §4.5 flow
- `impl AgentBackend for TmuxBackend { async fn spawn(...) -> Result<AgentHandle, CoreError> }`, mapping `BackendError → CoreError`.
- Naming (§4.3): `session = "agentd-<agent_id>"`, address `"<session>:0.0"`.
- Flow (§4.5): (1) `has-session` probe → if exists, `Recoverable` ("rebind instead"); (2) write `<worktree>/.agentd-launcher-<id>.sh` (cd/env/exec CLI) + amend `<worktree>/.gitignore` to exclude `.agentd-launcher-*.sh`; (3) build argv — Direct (`tmux new-session -d -s … -c … bash <launcher>`) vs Systemd (`systemd-run --user --scope … tmux new-session …`); (4) `display-message -p "#{pane_id} #{pane_pid}"` → parse `pane_id`/`pid`; (5) if `initial_prompt`, `wait_for_ready` + `send_prompt` (Task 3).
- Tests (FakeRunner): spawn succeeds + `pane_id` parsed from scripted `display-message` output **(case 1)**; spawn when `has-session` shows existing → `Recoverable` **(case 2)**; launcher script written + `.gitignore` amended **(advisor anchor)**.

### Task 3 — Prompt injection (buffer path) + wait_for_ready
- Inherent `send_prompt` (§4.6): `set-buffer <prompt>` (argv) when ≤64KiB else `set-buffer -` (stdin); `paste-buffer -p -t <addr> -d`; settle delay; `send-keys -t <addr> Enter`. NEVER `send-keys -l`.
- Inherent `wait_for_ready` (§4.7): capture loop, exponential backoff to 15s deadline, `main_prompt_visible` against `ready_patterns[cli]`.
- Tests: injection uses buffer path — assert `set-buffer`+`paste-buffer` present and the prompt text is NOT an arg of any `send-keys` call (negative assertion, no `send-keys -l`) **(case 3)**; large (>64KiB) prompt → argv `set-buffer -` **(case 4)**.

### Task 4 — capture + status
- Inherent `capture` (§4.8 helper): `capture-pane -p -t <addr>` (+ `-e` for ansi, `-S -<lines>`), returns the buffer string.
- Inherent `status` (§4.8): step 1 `display-message -p "#{pane_current_command}"` → bash/zsh/sh ⇒ `Starting`/`Gone` (disambiguate via capture); step 2 if CLI present, capture twice (800ms gap), diff ⇒ `Idle`/`Busy`.
- Tests: capture returns scripted buffer; status maps a shell `pane_current_command` to `Starting`/`Gone`; status maps identical/changed double-captures to `Idle`/`Busy`.

### Task 5 — shutdown + rebind
- Inherent `shutdown` (§4.9): **(1) archive `capture-pane` (5000 lines) to `opts.archive_to` + record `final_capture_sha` BEFORE any kill**; (2) graceful `/exit` via `send_prompt`, wait `graceful_timeout` (8s); (3) C-c ×2, wait 3s; (4) `kill-session`. Returns `ShutdownReport { method, final_capture_sha }`.
- Inherent `rebind` (§4.10): `has-session` → missing ⇒ `Ok(None)`; else re-probe `display-message` → reconstruct `AgentHandle`.
- Tests: shutdown records the `capture-pane` argv BEFORE any `kill-session`/`C-c` in the recorded call sequence **(case 7 + advisor anchor — ordering assertion)**; rebind on a missing session → `Ok(None)`; **missing session on an operation → `Recoverable` (case 5)**.

### Task 6 — acceptance sweep + adversarial review
- Confirm all seven §4.11 cases are covered across the test files; `cargo test -p agentd-tmux --no-fail-fast` green.
- Run the SQL/parity-style adversarial-verification workflow over the tmux argv construction + the buffer-path/ordering invariants.
- Fold findings; tag `v0.0.0-p0.3`; roadmap row → done.

## Outcome

Six tasks, strict TDD (spec → failing test → impl → clippy/fmt → check.sh →
cargo-deny → commit). Two adversarial-review workflows (spawn flow; full-backend
acceptance sweep) surfaced only test-completeness gaps — no correctness, safety,
forbidden-token, or D1/D2 defects — all folded. `AgentBackend` stayed spawn-only
in the tagged core (D1); the other six capabilities are inherent methods on
`TmuxBackend`. 5 specs / 43 scenarios / 45 FakeRunner tests, no real tmux server.

**Known item for the real-tmux phase (do not fix in v0):** the large-prompt
path emits `set-buffer -` with the payload on stdin, per the locked D4. Real
tmux reads stdin via `load-buffer -`, not `set-buffer -`; this is invisible
under the FakeRunner (argv-only) and must be reconciled when a phase wires a
real tmux server, so it is not silently inherited.
