# P0.9 — Capstone: production RunHost + daemon + live run start + kill-9 resume

> **For agentic workers:** TDD plan. Each task is RED → code → GREEN → gate → commit. agentd-core is FROZEN (D1) — do NOT edit `crates/agentd-core/**` (incl. its `tests/`). Scope is OFFLINE assembly + in-process contract tests; the real-agent/docker/SIGKILL/demo parts ship as a documented deployment checklist (user-confirmed).

**Goal:** Assemble the shipped P0.1–P0.8 pieces into a working `agentd` daemon: a production `RunHost` over a real `SqliteStore` + per-call `Engine`, the daemon bootstrap, live `agentctl run start`, the P0.7-deferred event emit point, and the kill-9 resume + idempotent-replay drill — all proven in-process with a scriptable agent over a real SQLite file.

**Architecture:** P0.9 is **assembly, not new core logic** (agentd-core frozen, D1 — verified: `Engine::deliver_event` already `load_checkpoint`s and resumes; `Checkpoint::{load,write_atomic,resume_guard}`, `Store::{write,load}_checkpoint` are pub; the events table already exists). The production `RunHost` lives in **`agentd-bin`** (the composition root — it may depend on store/tmux/mempal; `agentd-surface` must stay store-free per P0.7 D2). `agentd-bin` gains a `lib.rs` so the assembly is integration-testable; `main.rs` becomes the thin entrypoint. The test "agent" is a scriptable in-process driver (reads the park, synthesizes the matching `EngineEvent`, submits it through `mcp_server::dispatch`) over a real `SqliteStore` on a tempfile + `FakeBackend`/`FixedClock`/`MempalStub`/`RecordingCommandRunner`.

**Tech:** Rust 2024; agentd-core (frozen) engine/checkpoint; agentd-store `SqliteStore`; agentd-surface `RunHost`/`dispatch`/axum router; agentd-tmux `TmuxBackend`; agentd-mempal; clap; axum; tokio.

## Decisions

- **D1 — Scope = offline assembly + in-process contract tests + a deployment checklist; tag `v0.0.0-p0.9` (user-confirmed).** P0.9-done is redefined from the roadmap's "nightly real-agent e2e 5/5 + 90-second live demo" to **code-complete assembly, contract-tested in-process**, with the real-agent / real-tmux / rmcp-stdio / real-Matrix-mempal / actual-SIGKILL / live-demo parts delivered as `docs/p0.9-deployment-checklist.md` (code-ready, run on a real box). MVP is proven-by-test here.
- **D2 — agentd-core FROZEN, zero edits (verified).** The only "missing" primitive is a `SystemClock` (today only `FixedClock` exists); `Clock` is a pub trait (`now_unix()->i64`), so `agentd-bin` adds a ~5-line `SystemClock` **against** it — additive, in the binary. No migration (the `events` table + all 14 tables are in `0001_init.sql`).
- **D3 — Production `RunHost` in `agentd-bin` (NOT agentd-surface, which stays store-free per P0.7 D2).** The 5 methods over a real `SqliteStore` + per-call `Engine::new(graph, registry, Ports{&SqliteStore,&TmuxBackend,&MempalStub|client,&SystemClock,&runner}, sha)`: `deliver` = construct Engine + `deliver_event` + emit; `run_snapshot` = run row + `load_checkpoint`; `open_task` = the NEW forward read (D4) + `task_runs` row → `TaskAssignment`; `review_counts` = review_repo counts; `events_from` = `event_repo::read_from`. `agentd-bin` becomes lib+bin.
- **D4 — `open_task` needs a NEW forward store read (the one non-assembly piece; D1-safe).** `task_repo` has only the reverse `lookup_park_by_task_run`; `open_task(run_id, node_id)` must PRODUCE `task_run_id`. Add `task_repo::find_open_task_run(run_id, node_id) -> Option<TaskRunId+worktree>` (a `SELECT … WHERE run_id=? AND node_id=? AND completed_at IS NULL` — columns exist, **no migration**). agentd-store is not frozen.
- **D5 — `TaskAssignment` is partially populated (known gap, Q4; no migration).** `task_runs` persists only `worktree_path`; there are no `agent_id`/`spec_path`/`plan_path` columns. MVP populates `task_run_id` + `worktree`, the rest `None`. Surfaced as a known gap, NOT papered over with a migration (schema frozen).
- **D6 — Emit point: one row per STATE-CHANGING `RunProgress` (Q5).** After `execute`/`deliver_event` returns: `Parked`→`run_parked`, `Finished`→`run_finished`, `Failed`→`run_failed`; **exclude `Ignored`**; no node-level events. Payload is `serde_json::to_string` (COMPACT — avoids the P0.7 D9 SSE CR/LF hazard). One `event_repo::append` per state change.
- **D7 — Scriptable-agent harness + the `review_run_id` limitation (advisor).** The driver loops {observe park → synthesize event → submit via `mcp_server::dispatch`}. `dispatch("submit_outcome")` returns `{recorded, next_node}` and **swallows the `ParkReason`**, so the driver cannot learn `review_run_id` through the tool layer — for `execute.dot`'s fan-out it observes the in-process `RunProgress`/reads `review_repo`. Stated honestly: offline proves the engine/store/emit contract; "the reviewer learns WHAT to review" (spawn-context → rmcp) is the deferred real-env seam.
- **D8 — `agentctl run start` (live) = HTTP POST `/runs` to the daemon (Q1).** Daemon owns the store; agentctl stays thin (no store dep). The daemon resolves `workflows/<flow>.dot` (Q3: re-read by flow name + recompute sha; `resume_guard` gates sha-mismatch), `insert_run`, `engine.execute`, returns the initial `RunProgress`; agentctl prints + exits (user watches via SSE). Replaces the P0.8 "deferred to P0.9" error.
- **D9 — kill-9 drill = simulated restart + the SIGNATURE replay-safety assertion (advisor).** Resume-after-reopen is the easy half. The load-bearing checks: deliver the same event twice across a `drop(SqliteStore)`+reopen-same-file → second returns `Ignored`, NO duplicate `node_outcomes` row, event `seq` continuous; AND both `resume_guard` branches (sha-change rejected without `--accept-workflow-change`, accepted with). This is P0.9's analog of P0.8's goal_gate negative test.
- **D10 — rmcp stdio + real agents deferred (Q2/Q6).** `rmcp = "0.1"` is declared-but-unused (root Cargo.toml); offline drives the tool layer through the transport-agnostic `dispatch` seam. rmcp wire + version-bump, real tmux spawn, real mempal/Matrix, actual SIGKILL, nightly e2e + the 90s demo → the deployment checklist.

### 9d — kill-9 resume + replay drill

- **9d-T6 — the disaster-recovery drill (D9, the signature).** Spec **p92**. RED→GREEN over both workflows: execute to a Park; `drop(store)`; reopen same tempfile; fresh Engine; resolve the park via `dispatch` → assert (a) resumes past the parked node, (b) no completed node re-runs (`node_outcomes` count), (c) a new checkpoint exists, (d) event `seq` continuous across the boundary; **(e) replay-safety: the same event twice → second `Ignored`, no duplicate `node_outcomes` row**; **(f) `resume_guard`: sha-change rejected without accept, accepted with**. Gate + commit.
