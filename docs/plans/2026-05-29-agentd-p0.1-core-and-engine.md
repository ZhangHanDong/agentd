# P0.1 — Core Domain + Workflow Engine Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans. This phase builds the heart of the system — favor narrow PRs (one Task = one PR) so review stays tight.

**Goal:** Implement the workflow engine and its surrounding domain types in `agentd-core`, with no real I/O — backends, stores, and mempal are pluggable trait objects with in-memory fakes for tests. This phase makes `agentctl flow validate workflows/sample.dot` real, and proves an attractor-shaped traversal can run end-to-end with `parallel.fan_out` / `parallel.fan_in` / `wait.human` / `conditional` / `tool` / `codergen` handlers.

**Scenario count note:** This plan enumerates roughly 54 test selectors across Tasks 2–10. That's more than the design doc's §7.2 hand-wave estimate of "35 scenarios" — the discrepancy is just plan granularity vs. design rough-cut. The 240-round budget still holds because each scenario is small (≤ 5 rounds avg); the catch is that the roadmap's Σ row is now 209 instead of 187.

**Architecture:** Pure-domain `agentd-core` crate. Three trait seams: `AgentBackend` (provided by future P0.3), `Store` (provided by future P0.2), `MempalClient` (provided by future P0.4). All three have in-memory fakes in `agentd-core::test_support`. The engine drives nodes through a state machine that mirrors attractor's Parse→Transform→Validate→Initialize→Execute→Finalize phases. Checkpoint after every node. goal_gate evaluated before any terminal node fires.

**Tech Stack:** `agentd-core` deps — `tokio`, `thiserror`, `serde`, `serde_json`, `gv-rs`-equivalent DOT parser (we'll vendor a small one rather than take a heavy dep — see Task 1), `sha2`, `tracing`, `async-trait`, `futures`. Dev: `tempfile`, `tokio-test`.

**Spec:** Eight contracts under `specs/core/`:
- `p1-dot-parser.spec.md`
- `p2-node-graph-validate.spec.md`
- `p3-outcome-and-edge-selection.spec.md`
- `p4-checkpoint-and-resume.spec.md`
- `p5-goal-gate-enforcement.spec.md`
- `p6-handlers-conditional-and-tool.spec.md`
- `p7-handlers-fan-out-fan-in-wait-human.spec.md`
- `p8-engine-execute-loop.spec.md`

**Estimate:** 240 agent-rounds.

**Deps:** P0.0.

**Exit criteria:**

- `cargo test -p agentd-core` exits 0 with all 54 scenarios passing (delta-by-delta tally above).
- `agentctl flow validate workflows/sample.dot` works against a sample DOT shipped under `workflows/sample.dot`.
- CI gate "dot-validate" stops being a no-op stub: it runs `agentctl flow validate workflows/*.dot`.
- The engine can execute a 4-node DOT against fakes and reach a terminal node, persisting outcomes through the in-memory store.
- `agent-spec lifecycle 'specs/core/*.spec*' --min-score 0.7` passes for all eight specs.

## Task Pattern (canonical TDD loop, applies to every task below)

Each task follows the same shape:

1. Write/extend the relevant `.spec.md` with the scenario you're about to satisfy.
2. Write the failing test (function name = spec's `Test:` selector).
3. Run the test → confirm failure with the **expected** message (not just "any failure").
4. Implement the minimal code.
5. Run the test → confirm pass.
6. `cargo clippy --workspace --all-targets -- -D warnings` clean.
7. Commit with conventional message + `Refs: specs/core/p<N>-*.spec.md`.

Tasks 1 and 2 below are written out in full to anchor the pattern. Tasks 3–10 list deltas; expand them following the same shape.

```rust
//! Strongly-typed IDs. Newtype pattern — IDs of different domains never substitute.

use serde::{Deserialize, Serialize};
use std::fmt;

macro_rules! id_newtype {
    ($name:ident, $prefix:literal) => {
        #[derive(Debug, Clone, PartialEq, Eq, Hash, Serialize, Deserialize)]
        pub struct $name(String);

        impl $name {
            pub fn new() -> Self {
                Self(format!("{}_{}", $prefix, ulid::Ulid::new()))
            }

            pub fn from_string(s: impl Into<String>) -> Self {
                Self(s.into())
            }

            pub fn as_str(&self) -> &str {
                &self.0
            }
        }

        impl fmt::Display for $name {
            fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
                f.write_str(&self.0)
            }
        }

        impl Default for $name {
            fn default() -> Self {
                Self::new()
            }
        }
    };
}

id_newtype!(RunId, "r");
id_newtype!(TaskRunId, "tr");
id_newtype!(ReviewRunId, "rv");
id_newtype!(NodeId, "n");        // not auto-generated typically; parsed from DOT
id_newtype!(AgentId, "a");        // typically human-set ("claude-impl-a")

impl NodeId {
    /// NodeIds in DOT files are operator-authored; preserve as-is.
    pub fn parsed(s: impl Into<String>) -> Self {
        Self(s.into())
    }
}

impl AgentId {
    /// Agent IDs are operator-authored kebab-case strings.
    pub fn parsed(s: impl Into<String>) -> Self {
        Self(s.into())
    }
}
```

```rust
//! Shared workflow context. Persisted as JSON; merged by `context_updates` after each node.

use serde::{Deserialize, Serialize};

#[derive(Debug, Clone, Default, Serialize, Deserialize)]
pub struct RunContext(pub serde_json::Map<String, serde_json::Value>);

impl RunContext {
    pub fn new() -> Self {
        Self::default()
    }

    /// Apply a JSON merge patch (shallow).
    pub fn merge(&mut self, patch: &serde_json::Map<String, serde_json::Value>) {
        for (k, v) in patch {
            self.0.insert(k.clone(), v.clone());
        }
    }

    pub fn get(&self, key: &str) -> Option<&serde_json::Value> {
        self.0.get(key)
    }

    pub fn set(&mut self, key: impl Into<String>, value: serde_json::Value) {
        self.0.insert(key.into(), value);
    }
}
```

We are intentionally writing our own tiny DOT parser instead of pulling `graphviz-rs`-style deps. Rationale:

1. We use a strict subset (digraph, node-with-attrs, edge-with-attrs, comments). The full DOT grammar is overkill.
2. Avoiding the dep means CI build is faster and license surface is tighter.
3. The parser is ~250 LOC, the tests give us confidence.

```spec
## Intent

Parse `.agentflow/*.dot` files into a typed `dot::ast::Graph` containing
nodes (with attribute maps) and directed edges (with attribute maps).
The parser handles the subset of DOT we use: digraph keyword, named
nodes with `[attr=value, ...]` blocks, edges `"a" -> "b" [attr=...]`,
quoted and unquoted attribute values, `//` line comments, and trailing
commas/semicolons. Anything outside this subset is a parse error.

## Decisions

- Hand-written recursive parser, no external grammar deps
- Identifier characters: `a-z A-Z 0-9 _ - .`
- Attribute values are either bare identifiers, integers, booleans (true/false),
  double-quoted strings (with `\"` escape), or raw multi-line via `"""..."""`
- Comments are `// ... end-of-line` only; no `/* ... */`
- Node statements may appear standalone or implicitly through being referenced in an edge
- subgraph keyword reserved but rejected in v0 (error message points to roadmap)
- Order preservation: nodes and edges retain source order for deterministic iteration

## Boundaries

### Allowed Changes
- crates/agentd-core/src/dot/**
- crates/agentd-core/tests/dot_parser.rs

### Forbidden
- Do not depend on external DOT/graphviz crates.
- Do not silently accept tokens outside the documented subset.
- Do not normalize/lowercase identifiers.

## Completion Criteria

Scenario: Parse a minimal digraph with two nodes and an edge
  Test: dot_parser_minimal_digraph
  Given a DOT source:
    """
    digraph m {
      "a" [shape=Mdiamond];
      "b" [shape=Msquare];
      "a" -> "b";
    }
    """
  When the parser runs
  Then the result has 2 nodes with ids "a" and "b"
  And node "a" has attribute shape=Mdiamond
  And node "b" has attribute shape=Msquare
  And there is one edge from "a" to "b" with no attributes

Scenario: Node attributes accept quoted strings with metacharacters
  Test: dot_parser_quoted_attributes
  Given a DOT source declaring a node with attribute prompt="line 1\nline 2 with quote \""
  When the parser runs
  Then the node's prompt attribute equals the literal string with the embedded newline and quote

Scenario: Comments are stripped
  Test: dot_parser_strips_line_comments
  Given a DOT source with `// comment` lines mixed in
  When the parser runs
  Then the parsed graph is identical to the comment-free version

Scenario: Trailing commas and semicolons tolerated
  Test: dot_parser_trailing_punct
  Given a DOT source where every node and edge ends with a `;`
  And one node has a trailing `,` after the last attribute
  When the parser runs
  Then parsing succeeds

Scenario: Unknown subgraph keyword is rejected with a helpful error
  Test: dot_parser_subgraph_rejected
  Given a DOT source containing `subgraph cluster_0 { ... }`
  When the parser runs
  Then it returns Err
  And the error message contains "subgraph not supported"

  Test: dot_parser_edge_attributes
  Given a DOT source with `"a" -> "b" [condition="outcome=success", label="approve"];`
  When the parser runs
  Then the edge has condition="outcome=success" and label="approve"

Scenario: Empty graph yields zero nodes and zero edges
  Test: dot_parser_empty_graph
  Given `digraph empty { }`
  When the parser runs
  Then there are 0 nodes and 0 edges
  And the graph name is "empty"
```

```rust
//! Tests for `agentd_core::dot::parser`. Each test is named after the spec scenario's `Test:` selector.

use agentd_core::dot::{parser, ast::{Graph, Node, Edge}};
use pretty_assertions::assert_eq;

#[test]
fn dot_parser_minimal_digraph() {
    let src = r#"
        digraph m {
          "a" [shape=Mdiamond];
          "b" [shape=Msquare];
          "a" -> "b";
        }
    "#;
    let g = parser::parse(src).expect("parse");
    assert_eq!(g.name, "m");
    assert_eq!(g.nodes.len(), 2);
    assert_eq!(g.nodes[0].id, "a");
    assert_eq!(g.nodes[0].attrs.get("shape"), Some(&"Mdiamond".to_string()));
    assert_eq!(g.nodes[1].id, "b");
    assert_eq!(g.nodes[1].attrs.get("shape"), Some(&"Msquare".to_string()));
    assert_eq!(g.edges.len(), 1);
    assert_eq!(g.edges[0].from, "a");
    assert_eq!(g.edges[0].to, "b");
    assert!(g.edges[0].attrs.is_empty());
}

#[test]
fn dot_parser_quoted_attributes() {
    let src = r#"digraph m { "p" [prompt="line 1\nline 2 with quote \""]; }"#;
    let g = parser::parse(src).expect("parse");
    let prompt = g.nodes[0].attrs.get("prompt").expect("prompt attr");
    assert_eq!(prompt, "line 1\nline 2 with quote \"");
}

#[test]
fn dot_parser_strips_line_comments() {
    let with_comments = r#"
        digraph m {
          // this node is the start
          "a" [shape=Mdiamond];
          // this edge goes from a to b
          "a" -> "b";
          "b" [shape=Msquare];
        }
    "#;
    let without = r#"digraph m { "a" [shape=Mdiamond]; "a" -> "b"; "b" [shape=Msquare]; }"#;
    let g1 = parser::parse(with_comments).expect("parse with comments");
    let g2 = parser::parse(without).expect("parse without comments");
    assert_eq!(g1, g2);
}

#[test]
fn dot_parser_trailing_punct() {
    let src = r#"digraph m { "a" [shape=Mdiamond, ]; "a" -> "b"; "b"; }"#;
    parser::parse(src).expect("parse should tolerate trailing punctuation");
}

#[test]
fn dot_parser_subgraph_rejected() {
    let src = r#"digraph m { subgraph cluster_0 { "a"; } }"#;
    let err = parser::parse(src).expect_err("expected subgraph rejection");
    let msg = format!("{err:?}");
    assert!(
        msg.contains("subgraph not supported"),
        "error did not mention subgraph: {msg}"
    );
}

#[test]
fn dot_parser_edge_attributes() {
    let src = r#"digraph m { "a" -> "b" [condition="outcome=success", label="approve"]; }"#;
```

```rust
//! Hand-written parser for the DOT subset described in
//! `specs/core/p1-dot-parser.spec.md`.

use super::ast::{Edge, Graph, Node};
use crate::CoreError;
use std::collections::BTreeMap;

pub fn parse(src: &str) -> Result<Graph, CoreError> {
    let mut lexer = Lexer::new(src);
    parse_digraph(&mut lexer)
}

// ─── Lexer ──────────────────────────────────────────────────────────

struct Lexer<'a> {
    src: &'a str,
    pos: usize,
    peeked: Option<Tok>,
}

#[derive(Debug, Clone, PartialEq)]
enum Tok {
    Ident(String),
    QuotedString(String),
    Arrow,
    LBrace,
    RBrace,
    LBracket,
    RBracket,
    Eq,
    Comma,
    Semi,
    Eof,
}

impl<'a> Lexer<'a> {
    fn new(src: &'a str) -> Self { Self { src, pos: 0, peeked: None } }

    fn peek_tok(&mut self) -> Result<&Tok, CoreError> {
        if self.peeked.is_none() {
            self.peeked = Some(self.read_one()?);
        }
        // SAFETY: just populated.
        Ok(self.peeked.as_ref().expect("peeked just set"))
    }

    fn next(&mut self) -> Result<Tok, CoreError> {
        if let Some(t) = self.peeked.take() { return Ok(t); }
        self.read_one()
    }

    fn peek(&self) -> Option<char> {
        self.src[self.pos..].chars().next()
    }

    fn bump(&mut self) -> Option<char> {
        let c = self.peek()?;
        self.pos += c.len_utf8();
        Some(c)
    }

    fn skip_ws_and_comments(&mut self) {
        loop {
            // whitespace
            while let Some(c) = self.peek() {
                if c.is_whitespace() { self.bump(); } else { break; }
            }
            // line comment
            if self.src[self.pos..].starts_with("//") {
                while let Some(c) = self.bump() { if c == '\n' { break; } }
                continue;
            }
            break;
        }
    }

    fn read_one(&mut self) -> Result<Tok, CoreError> {
        self.skip_ws_and_comments();
        let Some(c) = self.peek() else { return Ok(Tok::Eof); };

        if c == '{' { self.bump(); return Ok(Tok::LBrace); }
        if c == '}' { self.bump(); return Ok(Tok::RBrace); }
        if c == '[' { self.bump(); return Ok(Tok::LBracket); }
        if c == ']' { self.bump(); return Ok(Tok::RBracket); }
        if c == '=' { self.bump(); return Ok(Tok::Eq); }
        if c == ',' { self.bump(); return Ok(Tok::Comma); }
        if c == ';' { self.bump(); return Ok(Tok::Semi); }

        if c == '-' {
            self.bump();
            if self.peek() == Some('>') { self.bump(); return Ok(Tok::Arrow); }
            return Err(CoreError::DotParse(format!("unexpected '-' at pos {}", self.pos)));
        }

        if c == '"' { return self.read_quoted(); }

        if c.is_ascii_alphanumeric() || c == '_' || c == '.' {
            return self.read_ident();
        }

        Err(CoreError::DotParse(format!("unexpected char {:?} at pos {}", c, self.pos)))
    }

    fn read_ident(&mut self) -> Result<Tok, CoreError> {
        let start = self.pos;
        while let Some(c) = self.peek() {
            if c.is_ascii_alphanumeric() || c == '_' || c == '-' || c == '.' {
                self.bump();
            } else { break; }
        }
        Ok(Tok::Ident(self.src[start..self.pos].to_string()))
    }

    fn read_quoted(&mut self) -> Result<Tok, CoreError> {
        self.bump(); // opening "
        let mut out = String::new();
        loop {
            let Some(c) = self.bump() else {
                return Err(CoreError::DotParse("unterminated quoted string".into()));
            };
            if c == '"' { return Ok(Tok::QuotedString(out)); }
            if c == '\\' {
                let Some(esc) = self.bump() else {
                    return Err(CoreError::DotParse("trailing backslash in quoted string".into()));
                };
                match esc {
                    'n' => out.push('\n'),
                    't' => out.push('\t'),
                    'r' => out.push('\r'),
                    '"' => out.push('"'),
                    '\\' => out.push('\\'),
                    other => return Err(CoreError::DotParse(format!("unknown escape \\{other}"))),
                }
            } else {
                out.push(c);
            }
        }
    }
}

// ─── Parser ─────────────────────────────────────────────────────────

fn parse_digraph(lex: &mut Lexer) -> Result<Graph, CoreError> {
    // expect: 'digraph'
    match lex.next()? {
        Tok::Ident(kw) if kw == "digraph" => {}
        other => return Err(CoreError::DotParse(format!("expected 'digraph', got {other:?}"))),
    }

    // optional name (may be elided: `digraph { ... }`)
    let name = match lex.peek_tok()? {
        Tok::LBrace => String::new(),
        _ => match lex.next()? {
            Tok::Ident(s) | Tok::QuotedString(s) => s,
            other => return Err(CoreError::DotParse(format!("expected graph name, got {other:?}"))),
        },
    };

    // require `{`
    match lex.next()? {
        Tok::LBrace => {}
        other => return Err(CoreError::DotParse(format!("expected '{{', got {other:?}"))),
    }

    let mut graph = Graph { name, ..Default::default() };

    loop {
        let tok = lex.next()?;
        match tok {
            Tok::RBrace => return Ok(graph),
            Tok::Eof => return Err(CoreError::DotParse("unexpected EOF inside graph".into())),
            Tok::Semi => continue,
            Tok::Ident(s) if s == "subgraph" => {
                return Err(CoreError::DotParse("subgraph not supported in v0; see roadmap".into()));
            }
            Tok::Ident(id) | Tok::QuotedString(id) => {
                parse_stmt_after_id(lex, &mut graph, id)?;
            }
            other => return Err(CoreError::DotParse(format!("unexpected token {other:?}"))),
        }
    }
}

fn parse_stmt_after_id(lex: &mut Lexer, graph: &mut Graph, id: String) -> Result<(), CoreError> {
    // peek decides node vs edge vs bare-id
    match lex.peek_tok()? {
        Tok::Arrow => {
            let _ = lex.next()?;                 // consume '->'
            let to = match lex.next()? {
                Tok::Ident(s) | Tok::QuotedString(s) => s,
                other => return Err(CoreError::DotParse(format!("expected target id after '->', got {other:?}"))),
            };
            let attrs = maybe_attrs(lex)?;
            graph.edges.push(Edge { from: id, to, attrs });
        }
        Tok::LBracket => {
            let _ = lex.next()?;                 // consume '['
            let attrs = parse_attr_list_body(lex)?;
            graph.nodes.push(Node { id, attrs });
        }
        _ => {
            graph.nodes.push(Node { id, attrs: BTreeMap::new() });
        }
    }
    consume_optional_semi(lex)?;
    Ok(())
}

fn maybe_attrs(lex: &mut Lexer) -> Result<BTreeMap<String, String>, CoreError> {
    if matches!(lex.peek_tok()?, Tok::LBracket) {
        let _ = lex.next()?;                     // consume '['
        parse_attr_list_body(lex)
    } else {
        Ok(BTreeMap::new())
    }
}

fn parse_attr_list_body(lex: &mut Lexer) -> Result<BTreeMap<String, String>, CoreError> {
    let mut attrs = BTreeMap::new();
    loop {
        let tok = lex.next()?;
        match tok {
            Tok::RBracket => return Ok(attrs),
            Tok::Comma => continue,
            Tok::Ident(k) | Tok::QuotedString(k) => {
                match lex.next()? {
                    Tok::Eq => {}
                    other => return Err(CoreError::DotParse(format!("expected '=' after attr name {k}, got {other:?}"))),
                }
                let v = match lex.next()? {
                    Tok::Ident(s) | Tok::QuotedString(s) => s,
                    other => return Err(CoreError::DotParse(format!("expected attr value, got {other:?}"))),
                };
                attrs.insert(k, v);
            }
            other => return Err(CoreError::DotParse(format!("unexpected token in attr list: {other:?}"))),
        }
    }
}

fn consume_optional_semi(lex: &mut Lexer) -> Result<(), CoreError> {
    if matches!(lex.peek_tok()?, Tok::Semi) {
        let _ = lex.next()?;
    }
    Ok(())
}
```
