## Engine Execution Model (READ FIRST — authoritative)

This section resolves the execution-model questions that earlier drafts left as
"TBD / decide during impl". **Every task below depends on these decisions; do not
deviate.** A self-review caught that the handler/engine contract was described two
incompatible ways (a synchronous "block until result" model for `codergen` vs. a
"park and resume via event" model for `wait.human`). The two cannot coexist. Here
is the single model.

### D1 — Handlers never block. They return a `HandlerStep`.

A handler's `run()` returns immediately with one of two outcomes:

```rust
// crates/agentd-core/src/engine/step.rs
pub enum HandlerStep {
    /// Handler finished synchronously (conditional, tool). Engine records the
    /// Outcome and selects the next edge right away.
    Done(Outcome),
    /// Handler started external work and cannot finish in this call. Engine
    /// persists a checkpoint, stops the run loop, and waits for a matching
    /// event delivered later via `Engine::deliver_event`.
    Park(ParkReason),
}

pub enum ParkReason {
    /// wait.human node — waits for a slash-command answer.
    /// `wait_id` is a plain String (the wait UUID), NOT a store newtype:
    /// agentd-core must not depend on agentd-store.
    HumanAnswer { wait_id: String },
    /// parallel.fan_out node — waits for all N reviewers to submit verdicts.
    ReviewVerdicts { review_run_id: ReviewRunId, expected: usize },
    /// codergen node — waits for the agent process to call submit_outcome.
    AgentOutcome { task_run_id: TaskRunId },
}
```

Rationale: `wait.human` may block for 72 hours — a handler that blocks would pin a
tokio task forever and break checkpoint/resume. So ALL long-running handlers park.
`conditional` and `tool` are the only synchronous handlers (they return `Done`).

### D2 — There is no `Status::Pending`.

Parking is an engine-level control-flow state, not a node Outcome. `Status` stays
exactly as defined in Task 1 (`Success | Fail | Retry | PartialSuccess`). A parked
node has *no* `node_outcomes` row yet — the row is written only when the park is
resolved and the handler produces its real `Outcome`. (Earlier drafts floated
adding `Status::Pending`; do not. Park lives in `HandlerStep::Park`.)

### D3 — Resumption: `Engine::deliver_event`.

```rust
pub enum EngineEvent {
    HumanAnswered { wait_id: String, answer: String, feedback: Option<String> },
    ReviewVerdictSubmitted { review_run_id: ReviewRunId, reviewer_id: AgentId },
    AgentOutcomeSubmitted { task_run_id: TaskRunId, outcome: Outcome },
}

impl Engine {
    /// Wake a parked run. Looks up which node parked on this ParkReason,
    /// re-invokes that handler's `resume(event) -> HandlerStep`, and continues
    /// the run loop if the handler now returns Done.
    pub async fn deliver_event(&self, event: EngineEvent) -> Result<RunProgress, CoreError>;
}
```

In P0.1 there is no MCP server or Matrix adapter yet, so tests synthesize these
events directly: a test calls `engine.deliver_event(EngineEvent::HumanAnswered{..})`
to simulate a mid-execution human decision (e.g. an operator answering an
`agent.blocked` prompt — boundary Δ2; spec approval itself lives in Specify, not
here). P0.6 (Matrix) and P0.7 (MCP server) later translate real-world signals into
these same events.

### D4 — Handler trait is `#[async_trait]` (dyn-compatible).

Native `async fn` in traits is NOT object-safe; `Arc<dyn Handler>` would not
compile. Every trait that is stored as `dyn` MUST use `#[async_trait::async_trait]`:
`Handler`, `AgentBackend`, `CommandRunner`, and the `ports::` traits (`Store`,
`MempalClient`). `Clock` is sync and does not need it. (`async-trait` is already a
Task 1 dependency.) The trait shape:

```rust
#[async_trait::async_trait]
pub trait Handler: Send + Sync {
    /// First execution of the node.
    async fn run(&self, ctx: &mut HandlerCtx<'_>) -> Result<HandlerStep, CoreError>;
    /// Re-execution after a Park was resolved. Default: unreachable for
    /// synchronous handlers (they never park).
    async fn resume(&self, _ctx: &mut HandlerCtx<'_>, _event: EngineEvent)
        -> Result<HandlerStep, CoreError> {
        Err(CoreError::Invariant("handler does not support resume".into()))
    }
}
```

### D5 — Port traits are defined BEFORE handlers (Task 6.5), not in Task 9.

`HandlerCtx` borrows `&dyn AgentBackend`, `&dyn Store`, `&dyn MempalClient`,
`&dyn Clock`, and `&dyn CommandRunner`. Those traits therefore must exist before
Task 7 (handlers) compiles. They are created in **Task 6.5** (new), which also
ships the in-memory fakes. Task 9 then only contains the engine run-loop +
`deliver_event` + the end-to-end engine tests.

### D6 — `CommandRunner` lives in `ports::command_runner` and is owned by agentd-core.

`tool` handler and (later) the tmux backend both shell out through one
`CommandRunner` trait. It is defined in `crates/agentd-core/src/ports/command_runner.rs`
in Task 6.5. P0.3's `TokioCommandRunner` implements it. The P0.1 fake
(`RecordingCommandRunner`) records argv and returns scripted outputs.

### D7 — `fan_out` in P0.1 does NOT materialize a ReviewBundle to disk.

P0.1 is "no real I/O". The full ReviewBundle directory materialization (MANIFEST,
context_sha, per-reviewer stance packs on disk) is deferred to **P0.6** when real
reviewers exist. In P0.1, `FanOutHandler::run`:
1. asks the (fake) backend to "spawn" N reviewer handles,
2. writes a `review_runs` row through the `Store` port (fake = in-memory),
3. returns `HandlerStep::Park(ParkReason::ReviewVerdicts { expected: N })`.

The `context_sha` is computed from in-memory context bytes (sha2 over the serialized
RunContext + node id), not from a filesystem MANIFEST. That keeps the determinism
property testable now without filesystem writes. P0.6 swaps the in-memory sha for
the real MANIFEST-based one behind the same `Store`/bundle port.

### D8 — Failure routing, goal_gate, retry, and the fan_out↔fan_in pairing rule.

This decision closes the control-flow holes around failure and pairing. Four parts:

**(a) goal_gate failure is an Outcome, not an error at the routing layer.**
Task 6's `goal_gate` check produces a *diagnostic*, but the engine converts an unmet
gate into a normal routable `Outcome` so edge selection (Task 4) works uniformly.
Concretely: when the engine is about to transition to a terminal node, it runs the
goal_gate check. If unmet, instead of bubbling `CoreError::GoalGateNotMet`, the
engine synthesizes `Outcome { status: Fail, preferred_label: Some("goal_gate_unmet"), .. }`
for the CURRENT node and re-runs edge selection from it. Edges authored as
`[condition="outcome=goal_gate_unmet"]` or carrying `retry_target=true` then route the
run back. `CoreError::GoalGateNotMet` is reserved for the case where NO such recovery
edge exists → the run ends `Failed`. (So Task 6 returns a `GoalGateStatus { met: bool,
missing: Vec<NodeId> }` to the engine; the engine, not goal_gate, decides routing.)

**(b) `retry_target` is a recognized edge attribute, parsed and honored in P0.1.**
Task 3 (validate) accepts `retry_target` (bool) as a known edge attribute. Task 4
(edge selection) adds it to the priority: when the current Outcome is `Fail` (or the
synthesized `goal_gate_unmet`), an outgoing edge with `retry_target=true` is preferred
over unconditional edges (slots between step 2 `preferred_label` and step 4 `weight`).
This is what lets `verify_lifecycle --fail--> implement [retry_target=true]` work.

**(c) per-node retry bound prevents infinite loops; `Status::Retry` is wired.**
Each node tracks an attempt counter (already in `Checkpoint.retry_counts`, Task 5).
The node attribute `retry_policy="max=N,..."` (design §2.2) sets the ceiling; default
`max=0` (no auto-retry). When a handler returns `Outcome{status: Retry, ..}`, the engine
re-runs the SAME node if `attempts < max`, else converts it to `Fail` and routes
normally. A `retry_target` edge that would re-enter a node already at its attempt
ceiling is NOT taken — the engine falls through to the next edge, and if none, ends
`Failed`. This guarantees termination. (So `Status::Retry` is neither dead code nor
infinite: it is the handler-requested same-node re-run, bounded by `retry_policy`.)

**(d) fan_out↔fan_in pairing rule.**
A `parallel.fan_in` node aggregates **the most-recent still-open `review_run` on the
current run** (looked up via `Store`). P0.1's validator (Task 3) enforces the simple
structural rule that makes this unambiguous: a fan_in node has exactly one fan_out
among its transitive predecessors with no other open review_run in between. The
canonical MVP DOT has exactly one fan_out→fan_in pair, so this is unambiguous now;
multi-fan_out graphs are rejected by `flow validate` until a later phase adds an
explicit `pair_with=<fan_out_node_id>` edge attribute.

## Task Pattern (canonical TDD loop, applies to every task below)

Each task follows the same shape:

1. Write/extend the relevant `.spec.md` with the scenario you're about to satisfy.
2. Write the failing test (function name = spec's `Test:` selector).
3. Run the test → confirm failure with the **expected** message (not just "any failure").
4. Implement the minimal code.
5. Run the test → confirm pass.
6. `cargo clippy --workspace --all-targets -- -D warnings` clean.
7. Commit with conventional message + `Refs: specs/core/p<N>-*.spec.md`.

Tasks 1 and 2 below are written out in full to anchor the pattern. Tasks 3–10 list deltas; expand them following the same shape.

//! Strongly-typed IDs. Newtype pattern — IDs of different domains never substitute.

use serde::{Deserialize, Serialize};
use std::fmt;

macro_rules! id_newtype {
    ($name:ident, $prefix:literal) => {
        #[derive(Debug, Clone, PartialEq, Eq, Hash, Serialize, Deserialize)]
        pub struct $name(String);

        impl $name {
            pub fn new() -> Self {
                Self(format!("{}_{}", $prefix, ulid::Ulid::new()))
            }

            pub fn from_string(s: impl Into<String>) -> Self {
                Self(s.into())
            }

            pub fn as_str(&self) -> &str {
                &self.0
            }
        }

        impl fmt::Display for $name {
            fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
                f.write_str(&self.0)
            }
        }

        impl Default for $name {
            fn default() -> Self {
                Self::new()
            }
        }
    };
}

id_newtype!(RunId, "r");
id_newtype!(TaskRunId, "tr");
id_newtype!(ReviewRunId, "rv");
id_newtype!(NodeId, "n");        // not auto-generated typically; parsed from DOT

        Self(s.into())
    }
}

impl AgentId {
    /// Agent IDs are operator-authored kebab-case strings.
    pub fn parsed(s: impl Into<String>) -> Self {
        Self(s.into())
    }
}

## Intent

Parse `.agentflow/*.dot` files into a typed `dot::ast::Graph` containing
nodes (with attribute maps) and directed edges (with attribute maps).
The parser handles the subset of DOT we use: digraph keyword, named
nodes with `[attr=value, ...]` blocks, edges `"a" -> "b" [attr=...]`,
quoted and unquoted attribute values, `//` line comments, and trailing
commas/semicolons. Anything outside this subset is a parse error.

## Decisions

- Hand-written recursive parser, no external grammar deps
- Identifier characters: `a-z A-Z 0-9 _ - .`
- Attribute values are either bare identifiers, integers, booleans (true/false),
  double-quoted strings (with `\"` escape), or raw multi-line via `"""..."""`
- Comments are `// ... end-of-line` only; no `/* ... */`
- Node statements may appear standalone or implicitly through being referenced in an edge
- subgraph keyword reserved but rejected in v0 (error message points to roadmap)
- Order preservation: nodes and edges retain source order for deterministic iteration

## Boundaries

### Allowed Changes
- crates/agentd-core/src/dot/**
- crates/agentd-core/tests/dot_parser.rs

### Forbidden
- Do not depend on external DOT/graphviz crates.
- Do not silently accept tokens outside the documented subset.
- Do not normalize/lowercase identifiers.

`crates/agentd-core/src/dot/parser.rs`:

```rust
//! Hand-written parser for the DOT subset described in
//! `specs/core/p1-dot-parser.spec.md`.

use super::ast::{Edge, Graph, Node};
use crate::CoreError;
use std::collections::BTreeMap;

pub fn parse(src: &str) -> Result<Graph, CoreError> {
    let mut lexer = Lexer::new(src);
    parse_digraph(&mut lexer)
}

// ─── Lexer ──────────────────────────────────────────────────────────

struct Lexer<'a> {
    src: &'a str,
    pos: usize,
    peeked: Option<Tok>,
}

#[derive(Debug, Clone, PartialEq)]
enum Tok {
    Ident(String),
    QuotedString(String),
    Arrow,
    LBrace,
    RBrace,
    LBracket,
    RBracket,
    Eq,
    Comma,
    Semi,
    Eof,
}

impl<'a> Lexer<'a> {
    fn new(src: &'a str) -> Self { Self { src, pos: 0, peeked: None } }

    fn peek_tok(&mut self) -> Result<&Tok, CoreError> {
        if self.peeked.is_none() {
            self.peeked = Some(self.read_one()?);
        }
        // SAFETY: just populated.
        Ok(self.peeked.as_ref().expect("peeked just set"))
    }

    fn next(&mut self) -> Result<Tok, CoreError> {
        if let Some(t) = self.peeked.take() { return Ok(t); }
        self.read_one()
    }

    fn peek(&self) -> Option<char> {
        self.src[self.pos..].chars().next()
    }

    fn bump(&mut self) -> Option<char> {
        let c = self.peek()?;
        self.pos += c.len_utf8();
        Some(c)
    }

    fn skip_ws_and_comments(&mut self) {
        loop {
            // whitespace
            while let Some(c) = self.peek() {
                if c.is_whitespace() { self.bump(); } else { break; }
            }
            // line comment
            if self.src[self.pos..].starts_with("//") {
                while let Some(c) = self.bump() { if c == '\n' { break; } }
                continue;
            }
            break;
        }
    }

    fn read_one(&mut self) -> Result<Tok, CoreError> {
        self.skip_ws_and_comments();
        let Some(c) = self.peek() else { return Ok(Tok::Eof); };

        if c == '{' { self.bump(); return Ok(Tok::LBrace); }
        if c == '}' { self.bump(); return Ok(Tok::RBrace); }
        if c == '[' { self.bump(); return Ok(Tok::LBracket); }
        if c == ']' { self.bump(); return Ok(Tok::RBracket); }
        if c == '=' { self.bump(); return Ok(Tok::Eq); }
        if c == ',' { self.bump(); return Ok(Tok::Comma); }
        if c == ';' { self.bump(); return Ok(Tok::Semi); }

        if c == '-' {
            self.bump();
            if self.peek() == Some('>') { self.bump(); return Ok(Tok::Arrow); }
            return Err(CoreError::DotParse(format!("unexpected '-' at pos {}", self.pos)));
        }

        if c == '"' { return self.read_quoted(); }

        if c.is_ascii_alphanumeric() || c == '_' || c == '.' {
            return self.read_ident();
        }

        Err(CoreError::DotParse(format!("unexpected char {:?} at pos {}", c, self.pos)))
    }

    fn read_ident(&mut self) -> Result<Tok, CoreError> {
        let start = self.pos;
        while let Some(c) = self.peek() {
            if c.is_ascii_alphanumeric() || c == '_' || c == '-' || c == '.' {
                self.bump();
            } else { break; }
        }
        Ok(Tok::Ident(self.src[start..self.pos].to_string()))
    }

    fn read_quoted(&mut self) -> Result<Tok, CoreError> {
        self.bump(); // opening "
        let mut out = String::new();
        loop {
            let Some(c) = self.bump() else {
                return Err(CoreError::DotParse("unterminated quoted string".into()));
            };
            if c == '"' { return Ok(Tok::QuotedString(out)); }
            if c == '\\' {
                let Some(esc) = self.bump() else {
                    return Err(CoreError::DotParse("trailing backslash in quoted string".into()));
                };
                match esc {
                    'n' => out.push('\n'),
                    't' => out.push('\t'),
                    'r' => out.push('\r'),
                    '"' => out.push('"'),
                    '\\' => out.push('\\'),
                    other => return Err(CoreError::DotParse(format!("unknown escape \\{other}"))),
                }
            } else {
                out.push(c);
            }
        }
    }
}

// ─── Parser ─────────────────────────────────────────────────────────

fn parse_digraph(lex: &mut Lexer) -> Result<Graph, CoreError> {
    // expect: 'digraph'
    match lex.next()? {
        Tok::Ident(kw) if kw == "digraph" => {}
        other => return Err(CoreError::DotParse(format!("expected 'digraph', got {other:?}"))),
    }

    // optional name (may be elided: `digraph { ... }`)
    let name = match lex.peek_tok()? {
        Tok::LBrace => String::new(),
        _ => match lex.next()? {
            Tok::Ident(s) | Tok::QuotedString(s) => s,
            other => return Err(CoreError::DotParse(format!("expected graph name, got {other:?}"))),
        },
    };

    // require `{`
    match lex.next()? {
        Tok::LBrace => {}
        other => return Err(CoreError::DotParse(format!("expected '{{', got {other:?}"))),
    }

    let mut graph = Graph { name, ..Default::default() };

    loop {
        let tok = lex.next()?;
        match tok {
            Tok::RBrace => return Ok(graph),
            Tok::Eof => return Err(CoreError::DotParse("unexpected EOF inside graph".into())),
            Tok::Semi => continue,
            Tok::Ident(s) if s == "subgraph" => {
                return Err(CoreError::DotParse("subgraph not supported in v0; see roadmap".into()));
            }
            Tok::Ident(id) | Tok::QuotedString(id) => {
                parse_stmt_after_id(lex, &mut graph, id)?;
            }
            other => return Err(CoreError::DotParse(format!("unexpected token {other:?}"))),
        }
    }
}

fn parse_stmt_after_id(lex: &mut Lexer, graph: &mut Graph, id: String) -> Result<(), CoreError> {
    // peek decides node vs edge vs bare-id
    match lex.peek_tok()? {
        Tok::Arrow => {
            let _ = lex.next()?;                 // consume '->'
            let to = match lex.next()? {
                Tok::Ident(s) | Tok::QuotedString(s) => s,
                other => return Err(CoreError::DotParse(format!("expected target id after '->', got {other:?}"))),
            };
            let attrs = maybe_attrs(lex)?;
            graph.edges.push(Edge { from: id, to, attrs });
        }
        Tok::LBracket => {
            let _ = lex.next()?;                 // consume '['
            let attrs = parse_attr_list_body(lex)?;
            graph.nodes.push(Node { id, attrs });
        }
        _ => {
            graph.nodes.push(Node { id, attrs: BTreeMap::new() });
        }
    }
    consume_optional_semi(lex)?;
    Ok(())
}

fn maybe_attrs(lex: &mut Lexer) -> Result<BTreeMap<String, String>, CoreError> {
    if matches!(lex.peek_tok()?, Tok::LBracket) {
        let _ = lex.next()?;                     // consume '['
        parse_attr_list_body(lex)
    } else {
        Ok(BTreeMap::new())
    }
}

fn parse_attr_list_body(lex: &mut Lexer) -> Result<BTreeMap<String, String>, CoreError> {
    let mut attrs = BTreeMap::new();
    loop {
        let tok = lex.next()?;
        match tok {
            Tok::RBracket => return Ok(attrs),
            Tok::Comma => continue,
            Tok::Ident(k) | Tok::QuotedString(k) => {
                match lex.next()? {
                    Tok::Eq => {}
                    other => return Err(CoreError::DotParse(format!("expected '=' after attr name {k}, got {other:?}"))),
                }
                let v = match lex.next()? {
                    Tok::Ident(s) | Tok::QuotedString(s) => s,
                    other => return Err(CoreError::DotParse(format!("expected attr value, got {other:?}"))),
                };
                attrs.insert(k, v);
            }
            other => return Err(CoreError::DotParse(format!("unexpected token in attr list: {other:?}"))),
        }
    }
}

fn consume_optional_semi(lex: &mut Lexer) -> Result<(), CoreError> {
    if matches!(lex.peek_tok()?, Tok::Semi) {
        let _ = lex.next()?;
    }
    Ok(())
}
```
